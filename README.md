# Електронний журнал університету

Django + PostgreSQL. Дозволяє швидко вносити студентів, дисципліни, спеціальності
й видавати викладачам точкові дозволи на конкретний журнал (дисципліна + група + семестр).

## Як влаштована система прав

Ключова модель — `journal.TeacherAssignment`. Це і є "дозвіл": деканат/адміністратор
вказує, якому викладачу яку дисципліну в якій групі й семестрі можна вести.

- Викладач бачить і редагує **лише** ті журнали, на які має активний `TeacherAssignment`
  — і в кабінеті (`/`), і в адмінці (`/admin/`). Підміна ID в URL не допоможе:
  усі вибірки проходять через `journal.permissions.get_teacher_assignments()`,
  тож запис поза межами дозволу просто не існує для запиту цього користувача (404).
- Деканат (`role=dean`) і адміністратор (`role=admin`/суперкористувач) бачать усе,
  керують структурою (факультети, спеціальності, групи, студенти, дисципліни)
  і **лише вони** можуть створювати/змінювати `TeacherAssignment` — самі викладачі
  не можуть видати собі доступ до чужого предмета.
- Студент (`role=student`) бачить тільки власні оцінки й відвідуваність.

Наприклад: викладач "Основ програмування" отримує `TeacherAssignment(discipline=Основи
програмування, group=ІПЗ-101)` і фізично не може відкрити журнал "Вищої математики"
чи будь-якої іншої групи.

## Запуск через Docker (найпростіше)

```bash
cp .env.example .env
docker compose up --build
```

Це підніме PostgreSQL, застосує міграції, наповнить демо-даними і запустить сервер
на http://localhost:8000/

## Запуск без Docker

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# відредагуйте .env, якщо PostgreSQL налаштований інакше

# створіть базу в PostgreSQL заздалегідь:
#   createdb university_journal
#   (або через psql: CREATE DATABASE university_journal; CREATE USER journal_user ...)

python manage.py makemigrations accounts university journal
python manage.py migrate
python manage.py seed_demo   # демонстраційні дані (необов'язково)
python manage.py runserver
```

## Демо-доступи (після `seed_demo`)

| Роль                | Логін         | Пароль        | Що бачить |
|---------------------|---------------|---------------|-----------|
| Адміністратор        | `admin`       | `admin12345`  | Усе, `/admin/` |
| Деканат              | `dean`        | `dean12345`   | Усю структуру + видача дозволів |
| Викладач програмування | `teacher_prog` | `teacher12345` | **ЛИШЕ** "Основи програмування" / ІПЗ-101 |
| Викладач математики  | `teacher_math` | `teacher12345` | **ЛИШЕ** "Вища математика" / ІПЗ-101 |
| Студенти             | `student1`…`student5` | `student12345` | Власні оцінки |

Увійдіть як `teacher_prog` і `teacher_math` по черзі, щоб побачити, що кожен
викладач має доступ тільки до свого предмета.

## Швидке внесення даних

- `/admin/` — деканат/адмін тут швидко додають факультети, спеціальності, групи,
  студентів (інлайн прямо в картці групи), дисципліни, навчальний план і головне —
  видають викладачам `TeacherAssignment` (дозволи).
- Кабінет викладача (`/`) — власний спрощений інтерфейс: додати заняття одним
  рядком, виставити оцінки й відвідуваність усій групі однією формою, зведена
  таблиця оцінок «студенти × заняття».

## Структура проєкту

```
accounts/    — кастомний User з ролями (admin/dean/teacher/student)
university/  — факультети, спеціальності, групи, студенти, дисципліни, навч. план
journal/     — дозволи викладачів (TeacherAssignment), заняття, оцінки, відвідуваність
config/      — settings/urls Django-проєкту
templates/   — Bootstrap-шаблони кабінету викладача/студента
```

## Що можна розширити далі

- Експорт відомостей у Excel/PDF

- REST API (Django REST Framework) для мобільного застосунку
